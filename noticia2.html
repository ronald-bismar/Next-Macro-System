¿Qué son las bases de datos NoSQL y por qué incluso el creador de SQL las apoya?

El creador de SQL, Donald Chamberlin, ha dado su respaldo a las bases de datos NoSQL, una categoría diversa de bases de datos que ha surgido en los últimos años y que se separa de su propia creación exitosa.

En la celebración del 50 aniversario del lenguaje SQL (Structured Query Language), Chamberlin, quien junto a Raymond Boyce desarrolló SQL en 1974, ha reconocido la importancia de las bases de datos NoSQL para aplicaciones modernas que requieren mayor escalabilidad y rendimiento. Aunque sigue valorando el papel fundamental de SQL, Chamberlin entiende que las nuevas aplicaciones, especialmente las web, necesitan bases de datos que soporten gran escalabilidad y alto rendimiento, algo que las bases de datos relacionales tradicionales no pueden proporcionar.

SQL vs NoSQL



SQL fue desarrollado como un lenguaje estandarizado para consultas en bases de datos relacionales, organizando los datos en tablas interconectadas con filas y columnas. Esta estructura facilitaba la gestión y el análisis de grandes volúmenes de datos, pero presentaba limitaciones en cuanto a flexibilidad y escalabilidad.

En 2009, Johan Oskarsson de Last.fm resucitó el término 'NoSQL' para describir alternativas a las bases de datos SQL tradicionales, las cuales no podían satisfacer las necesidades de muchas aplicaciones modernas, especialmente aquellas que manejan grandes volúmenes de datos y requieren alta escalabilidad.

A diferencia de SQL, las bases de datos NoSQL (Not Only SQL) ofrecen una solución más flexible, adoptando formatos como bases de datos de documentos, de valores clave y de grafos, lo que permite un manejo de datos más dinámico y adaptable a diferentes tipos de consultas y estructuras de datos. Por ejemplo, MongoDB y Couchbase utilizan JSON para almacenar datos de manera esquemática y flexible, facilitando la integración con aplicaciones web. Por otro lado, Redis y Neo4j proporcionan soluciones rápidas para el almacenamiento en caché y la representación de redes complejas de relaciones.

SQL vs NoSQL



Además de apoyar NoSQL, Chamberlin también promueve SQL++, un lenguaje de consulta desarrollado por el profesor Yannis Papakonstantinou de UC San Diego. SQL++ está diseñado para trabajar con datos semiestructurados o no estructurados, como los documentos JSON. Este nuevo lenguaje, adoptado por Couchbase y AWS bajo el nombre PartiQL, busca combinar las ventajas de NoSQL con la familiaridad y robustez de SQL.

Aunque Chamberlin está semiretirado, sigue trabajando como asesor técnico en Couchbase y enfatiza que, a pesar de los avances de NoSQL y SQL++, el SQL tradicional continuará siendo crucial en el manejo de datos empresariales debido a su eficacia, amplia adopción y múltiples implementaciones de código abierto altamente confiables.

"No creo que SQL vaya a desaparecer. Una gran parte de los datos empresariales del mundo están codificados en SQL, y los datos son muy persistentes. Una vez que tienes tu base de datos, te quedarás con ella".
